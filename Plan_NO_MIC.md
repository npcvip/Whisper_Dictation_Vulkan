# План: Обработка отсутствия микрофона + Окно распознавания аудиофайлов

## Часть 1: Обработка отсутствия микрофона

### 1.1. Проверка микрофона при запуске
- **Где:** `AudioRecorder.__init__()` (строка ~151)
- **Что изменить:** обернуть `self.p.get_default_input_device_info()` в try/except
- **Если микрофон найден:** работаем как обычно
- **Если не найден:**
  - Установить флаг `self.mic_available = False`
  - Установить `self.device_index = None`
  - Логировать предупреждение
  - Показать уведомление: "Микрофон не найден. Программа в режиме паузы. Нажмите правую кнопку мыши на иконке для настроек."

### 1.2. Уведомление о режиме паузы
- **Где:** `TrayIcon.__init__()`
- **Что изменить:** при инициализации проверить `core.audio.mic_available`
- **Если нет микрофона:**
  - Установить `self.paused = True`
  - Установить `self.hotkey_manager.paused = True`
  - Показать иконку как неактивную (серый круг)
  - Показать уведомление (тоаст) на 3 сек

### 1.3. Проверка микрофона при снятии с паузы
- **Где:** `TrayIcon._toggle_pause()` и `TrayIcon._on_click()`
- **Что изменить:** перед снятием с паузы проверить микрофон
- **Логика:**
  - Если `paused == True` и пользователь пытается выключить паузу:
    1. Проверить микрофон через `pyaudio.PyAudio().get_default_input_device_info()`
    2. **Микрофон найден:**
       - Установить `self.paused = False`
       - Установить `self.hotkey_manager.paused = False`
       - Показать уведомление: "Микрофон найден! Программа готова к работе. Нажмите среднюю кнопку мыши для начала диктовки." (2 сек)
       - Обновить иконку (синий круг)
    3. **Микрофон не найден:**
       - Показать уведомление: "Подключите микрофон или выберите его по умолчанию в настройках Windows." (2 сек)
       - Остаться в режиме паузы

### 1.4. Уведомления (тоасты)
- **Где:** новый метод `TrayIcon._show_toast(title, message, duration=2)`
- **Что использовать:** `win10toast` или простое tkinter окно
- **Формат:** окно 300x80, зелёный фон, белый текст, автозакрытие через N сек
- **Альтернатива:** использовать `_show_temp_notification` из `SettingsWindow` как шаблон

### 1.5. Защита записи от запуска без микрофона
- **Где:** `DictationCore.start_recording()`
- **Что изменить:** проверить `self.audio.mic_available` перед запуском
- **Если нет:** не запускать запись, показать уведомление

## Часть 2: Окно "Выбрать аудиофайл"

### 2.1. Пункт в меню трея
- **Где:** `TrayIcon._setup_menu()`
- **Что добавить:** `pystray.MenuItem("Выбрать аудиофайл", self._on_audio_file)`

### 2.2. Класс окна `AudioFileWindow`
- **Где:** новый класс после `SettingsWindow`
- **Размер:** 600x500, фиксированный
- **Компоненты:**

#### Верхняя панель (row 0-1):
- **row 0:** `ttk.Label("Аудиофайл:")` + `ttk.Entry(self.file_path_var, width=60)` + `ttk.Button("Выбрать файл", command=self._browse_file)`
- **row 1:** `ttk.Button("Распознать", command=self._transcribe_file)` (широкая кнопка, зелёный цвет)

#### Текстовое поле (row 2):
- `tk.Text(self.text_var, wrap="word", state="disabled")` + `ttk.Scrollbar`
- Высота: ~15 строк

#### Кнопки внизу (row 3):
- `ttk.Button("Скопировать всё", command=self._copy_all)` + `ttk.Button("Очистить", command=self._clear_text)`

### 2.3. Методы `AudioFileWindow`

#### `_browse_file()`
- `filedialog.askopenfilename(filetypes=[("Аудиофайлы", "*.wav *.mp3 *.flac *.ogg"), ("Все файлы", "*.*")])`
- Записать путь в `self.file_path_var`

#### `_transcribe_file()`
- Получить путь из `self.file_path_var`
- Проверить, что файл существует
- Вызвать `self.runner.transcribe(file_path)`
- Добавить результат в текстовое поле:
  - Если текст уже есть: добавить `\n---\n` + новый текст
  - Если пусто: просто добавить текст
- Показать статус "Распознано" / "Ошибка"

#### `_copy_all()`
- `pyperclip.copy(self.text_var.get("1.0", "end-1c"))`

#### `_clear_text()`
- Очистить текстовое поле

### 2.4. Связь с `DictationCore`
- **Где:** `TrayIcon._on_audio_file()`
- **Что сделать:** создать `AudioFileWindow`, передать `core.runner` для транскрипции
- **Важно:** окно должно быть `Toplevel`, не закрывать при закрытии основного окна настроек
